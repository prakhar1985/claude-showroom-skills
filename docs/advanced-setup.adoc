== Prerequisites

This section is for you if you already have:

* Existing `~/.claude/` directory with custom configurations
* Other Claude Code skills installed
* Custom prompts, hooks, or settings.json
* Any setup you want to preserve

If you're new to Claude Code with no existing `~/.claude/` directory, use the link:setup.html[Standard Installation] instead.

== Step 1: Backup Your Configuration

[source,bash]
----
# ALWAYS backup first!
cp -r ~/.claude ~/.claude.backup
----

== Step 2: Copy Skills Individually

[source,bash]
----
# Navigate to nookbag template
cd /path/to/showroom-template-nookbag

# Create skills directory if it doesn't exist
mkdir -p ~/.claude/skills

# Copy each Showroom skill individually (no overwrite)
cp -r .claude/skills/create-lab ~/.claude/skills/
cp -r .claude/skills/create-demo ~/.claude/skills/
cp -r .claude/skills/verify-content ~/.claude/skills/
cp -r .claude/skills/blog-generate ~/.claude/skills/
cp -r .claude/skills/generate-agv-description ~/.claude/skills/
----

== Step 3: Copy Required Prompts

[source,bash]
----
# Create prompts directory if it doesn't exist
mkdir -p ~/.claude/prompts

# Copy Showroom prompts (these have unique names - won't conflict with custom prompts)
cp .claude/prompts/* ~/.claude/prompts/
----

[NOTE]
====
The `verify-content` skill requires all 11 prompt files to function. These prompts have unique Showroom-specific names and won't overwrite your custom prompts.
====

== Step 4: Optionally Copy Docs and Templates

[source,bash]
----
# Optional: Copy shared documentation
mkdir -p ~/.claude/docs
cp -r .claude/docs/* ~/.claude/docs/

# Optional: Copy reference templates
mkdir -p ~/.claude/templates
cp -r .claude/templates/* ~/.claude/templates/
----

== Step 5: Verify Coexistence

[source,bash]
----
# Start Claude Code
claude

# Check that BOTH your existing skills AND Showroom skills appear
/help

# You should see:
# - Your existing custom skills
# - Showroom skills: /verify-content, /create-lab, /create-demo, etc.
----

== Configuration Coexistence

[cols="1,3"]
|===
|Configuration |Behavior

|*settings.json*
|Showroom skills don't require special settings. Your existing `settings.json` works as-is.

|*Other skills*
|Multiple skills coexist peacefully. No conflicts between Showroom skills and your custom skills.

|*Custom prompts*
|Showroom prompts use unique names (`enhanced_verification_workshop.txt`, etc.). They won't overwrite your custom prompts.

|*Hooks*
|If you have custom hooks in `~/.claude/hooks/`, review Showroom hooks before copying. Merge manually if needed.

|*Agents*
|Showroom includes 8 AI validation agents. These are optional and won't interfere with existing setup.
|===

== Rollback If Needed

If something breaks after installation:

[source,bash]
----
# Remove new installation
rm -rf ~/.claude

# Restore your backup
mv ~/.claude.backup ~/.claude

# Restart Claude Code
exit
claude
----
