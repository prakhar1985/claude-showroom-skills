== Prerequisites

These skills work with Claude Code CLI or Cursor IDE. If you already have either tool set up on your laptop, you're ready to install Showroom Skills.

You'll also need:

* *Git* command line tools
* *Terminal* access (macOS Terminal, Linux shell, or Windows WSL)
* *Basic command line* familiarity

== Installation Steps

=== Step 1: Clone the Nookbag Template

The nookbag template repository contains all the skills, prompts, and templates you need.

[source,bash]
----
git clone https://github.com/rhpds/showroom_template_nookbag.git
cd showroom_template_nookbag
----

*What this gives you:*

* 4 core skills (`create-lab`, `create-demo`, `verify-content`, `blog-generate`)
* 11 verification prompt files
* Shared rules and documentation
* Workshop and demo templates
* Validation hooks

=== Step 2: Copy Skills to Your Home Directory

Skills need to be in your home directory (`~/.claude/`) to work globally across all projects.

==== Option A: Copy Everything (Recommended)

[source,bash]
----
# Copy the entire .claude directory to your home
cp -r .claude ~/.claude
----

This copies:

* `~/.claude/skills/` - All 4 skills
* `~/.claude/prompts/` - All 11 verification prompts
* `~/.claude/docs/` - Shared rules (SKILL-COMMON-RULES.md)
* `~/.claude/hooks/` - Validation hooks
* `~/.claude/agents/` - AI validation agents
* `~/.claude/templates/` - Reference templates

==== Option B: Copy Individual Components

If you already have a `.claude/` directory, copy selectively:

[source,bash]
----
# Copy skills
cp -r .claude/skills ~/.claude/

# Copy prompts (required for verify-content)
cp -r .claude/prompts ~/.claude/

# Copy shared documentation
cp -r .claude/docs ~/.claude/
----

[WARNING]
====
*Important:* The `verify-content` skill requires all files in `.claude/prompts/` to function correctly. Don't skip copying prompts!
====

=== Step 3: Verify Installation

Test that skills are properly installed and discovered by Claude Code.

[source,bash]
----
# Navigate to any directory
cd ~

# Start Claude Code
claude

# In the Claude Code session, type:
/help
----

*Expected output:*

You should see the Showroom skills listed:

----
Available custom skills:
  /verify-content    Validate workshop/demo content quality
  /create-lab        Create workshop module with guided workflow
  /create-demo       Create demo module with Know/Show structure
  /blog-generate     Transform content to blog posts
----

[TIP]
====
If skills don't appear, check the link:troubleshooting.adoc[Troubleshooting] page for common issues.
====

│   └── blog-generate/

│   ├── enhanced_verification_workshop.txt

│   ├── verify_accessibility_compliance_demo.txt

== Using Cursor IDE (Without Claude Code CLI)

If you use Cursor IDE and don't have Claude Code CLI installed, you can still use all Showroom skills with natural language commands.

=== Prerequisites for Cursor Users

* *Cursor IDE* installed (download from link:https://cursor.sh[cursor.sh^])
* *Git* command line tools
* *Basic familiarity* with Cursor's chat interface

=== Installation Steps for Cursor

==== Step 1: Clone the Nookbag Template

[source,bash]
----
git clone https://github.com/rhpds/showroom_template_nookbag.git
cd showroom_template_nookbag
----

==== Step 2: Open in Cursor IDE

[source,bash]
----
# Open the nookbag template in Cursor
cursor .
----

Or use Cursor's File → Open Folder menu.

==== Step 3: Skills Auto-Load

When you open the repository in Cursor, skills automatically load from `.cursor/rules/` directory. No manual configuration needed.

**What gets loaded:**

* `.cursor/rules/create-lab/RULE.md` - Workshop creation skill
* `.cursor/rules/create-demo/RULE.md` - Demo creation skill
* `.cursor/rules/verify-content/RULE.md` - Content verification skill
* `.cursor/rules/generate-agv-description/RULE.md` - AgV catalog generation

=== How to Use Skills in Cursor

Unlike Claude Code CLI (which uses slash commands like `/verify-content`), Cursor uses **natural language triggers** in the chat.

==== Available Commands

[cols="1,2,2"]
|===
|Skill |Cursor Chat Commands |What It Does

|*Verify Content*
|"verify content" +
"check quality" +
"validate workshop"
|Validates content against Red Hat standards

|*Create Lab*
|"create lab" +
"generate workshop module" +
"new workshop"
|Creates workshop module with guided questions

|*Create Demo*
|"create demo" +
"presenter-led demo" +
"generate demo module"
|Creates presenter demo with Know/Show structure

|*Generate AgV Description*
|"generate agv description" +
"create catalog description"
|Generates AgnosticV catalog description file
|===

=== Example Workflow in Cursor

**Creating a New Workshop:**

1. Open Cursor chat (Cmd+L or Ctrl+L)
2. Type: `create lab`
3. Claude will ask questions one at a time
4. Answer each question and press Enter
5. Claude generates files automatically

**Verifying Content:**

1. Open Cursor chat
2. Type: `verify content`
3. Claude checks all .adoc files and reports issues
4. Fix issues in the editor
5. Type `verify content` again to re-check

[IMPORTANT]
====
**Key Differences from Claude Code CLI:**

* *No slash commands* - Use natural language: "create lab" NOT `/create-lab`
* *Same workflow* - Questions, answers, and file generation work identically
* *Auto-loading* - Skills load automatically when you open the project
* *Project-specific* - Skills are tied to the project, not global like Claude Code CLI
====

=== Using Skills in Other Projects

[IMPORTANT]
====
Unlike Claude Code CLI, Cursor does **NOT support global rules** in a home directory. Skills must be in each project's `.cursor/rules/` directory.
====

To use Showroom skills in a new project:

[source,bash]
----
# Navigate to your new Showroom project
cd ~/my-new-showroom-project

# Copy .cursor rules from nookbag template
cp -r /path/to/showroom_template_nookbag/.cursor .

# Copy .claude directory as well (contains shared rules and prompts)
cp -r /path/to/showroom_template_nookbag/.claude .

# Open in Cursor
cursor .
----

**Alternative:** Use the nookbag template as your starting point for every new Showroom project:

[source,bash]
----
# Clone nookbag template with a new name
git clone https://github.com/rhpds/showroom_template_nookbag.git my-new-workshop
cd my-new-workshop

# Remove git history and start fresh
rm -rf .git
git init

# Open in Cursor - skills are already configured
cursor .
----

== Updating Skills

To update skills when the nookbag template is updated:

[source,bash]
----
# Navigate to nookbag template
cd path/to/showroom_template_nookbag

# Pull latest changes
git pull origin main

# Re-copy to home directory
cp -r .claude ~/.claude

# Restart Claude Code session
----

[WARNING]
====
Copying will overwrite your existing `~/.claude/` files. Back up any custom modifications first.
====

== System Requirements Details

=== Supported Operating Systems

* macOS (Intel and Apple Silicon)
* Linux (Ubuntu, Fedora, RHEL, Debian)
* Windows (via WSL2)

=== Claude Code Versions

* Minimum: Claude Code v1.0+
* Recommended: Latest version from link:https://code.claude.com[code.claude.com^]

=== Disk Space

* Skills: ~5 MB
* Prompts and docs: ~2 MB
* Total: ~7 MB

=== Network Requirements

* Internet connection for Claude API calls
* No additional network configuration needed
